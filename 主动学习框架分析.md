# 主动学习框架详细分析

基于 `demo3.ipynb` 的主动学习实现，从四个核心维度进行深入剖析。

**摘要**

我们采用了一种基于委员会查询的主动学习策略 (Committee-based Active Learning)，通过集成三个异构模型 (Random Forest, XGBoost, LightGBM) 的预测分歧来量化认知不确定性。虽然该方法并非严格意义上的贝叶斯优化，但其采集函数设计借鉴了贝叶斯思想中对模型不确定性的显式建模理念。同时，我们引入了物理先验引导的边界加权机制和K-Means 多样性采样，形成了一种混合主动学习框架。
## 1️⃣ 委员会不确定性方法 (Committee Uncertainty Method)

### 核心实现代码 (第 1020-1028 行)

```python
def committee_uncertainty(models, X_candidates):
    preds = [m.predict(X_candidates.values) for m in models]  # list of (n, n_targets)
    P = np.stack(preds, axis=0)                               # (n_models, n, n_targets)
    mu = P.mean(axis=2, keepdims=True)
    sd = P.std(axis=2, keepdims=True) + 1e-8
    Pz = (P - mu) / sd
    var = Pz.var(axis=0)                                      # (n, n_targets)
    w = TARGET_WEIGHTS[None, :]
    return (var * w).sum(axis=1)                              # (n,)
```

### 算法机制深度解析

#### 步骤 1: 多模型预测集成
```python
preds = [m.predict(X_candidates.values) for m in models]
P = np.stack(preds, axis=0)  # 形状：(n_models, n_samples, n_targets)
```
- **输入**: 3 个模型 (RandomForest, XGBoost, LightGBM)
- **输出**: 预测张量 P，包含每个模型对每个样本、每个目标的预测值
- **物理意义**: 捕捉不同模型对同一样本的理解差异

#### 步骤 2: 标准化处理 (Z-score Normalization)
```python
mu = P.mean(axis=2, keepdims=True)      # 跨目标求均值 (n_models, n, 1)
sd = P.std(axis=2, keepdims=True) + 1e-8  # 跨目标求标准差
Pz = (P - mu) / sd                      # 标准化后的预测 (n_models, n, n_targets)
```
- **关键技巧**: 在**目标维度 (axis=2)** 上进行标准化，消除不同目标的量纲影响
- **数学表达**: 
  $$
  Pz_{m,i,t} = \frac{P_{m,i,t} - \mu_{m,i}}{\sigma_{m,i} + \epsilon}
  $$
  其中 $\mu_{m,i} = \frac{1}{T}\sum_{t} P_{m,i,t}$

#### 步骤 3: 计算方差 (不确定性度量)
```python
var = Pz.var(axis=0)  # (n_samples, n_targets)
```
- **统计意义**: 计算 3 个模型标准化预测值的**样本方差**
- **公式**: 
  $$
  \text{var}_{i,t} = \frac{1}{M}\sum_{m=1}^{M}(Pz_{m,i,t} - \bar{Pz}_{i,t})^2
  $$
- **物理解释**: 方差越大 → 模型间分歧越大 → 该样本越"不确定"

#### 步骤 4: 加权聚合 (多目标融合)
```python
w = TARGET_WEIGHTS[None, :]  # [0.5, 0.3, 0.2]
return (var * w).sum(axis=1)  # 加权求和，返回 (n,)
```
- **权重分配策略**:
  - 致密度 (Density): **0.5** (最重要)
  - 阻抗 (Impedance): **0.3** (次重要)
  - Jcorr: **0.2** (再次要)
- **最终分数**: 
  $$
  \text{uncertainty}_i = 0.5 \cdot \text{var}_{i,\text{密度}} + 0.3 \cdot \text{var}_{i,\text{阻抗}} + 0.2 \cdot \text{var}_{i,\text{Jcorr}}
  $$

### 优势分析
✅ **多模型投票机制**: 利用 3 个异构模型的差异性，比单模型不确定性更可靠  
✅ **多目标协同**: 通过加权融合 3 个输出的不确定性，避免单一指标偏差  
✅ **标准化预处理**: 消除量纲影响，使不同目标的不确定性具有可比性  
✅ **数值稳定性**: `+1e-8` 防止除零错误

## 2️⃣ 边界加权机制 (Boundary Weighting Mechanism)

### 核心实现代码 (第 1030-1043 行)

```python
def boundary_weight(candidate_df):
    ed = candidate_df["能量密度"].values
    binlab = candidate_df["ED_bin"].values
    w = np.ones(len(candidate_df), dtype=float)
    
    # 中段边界附近 (76-84, 97-103)
    near_76 = (binlab == "Mid") & (np.abs(ed - 80) <= 3)
    near_100 = (binlab == "Mid") & (np.abs(ed - 110) <= 3)
    w[near_76] *= 1.3
    w[near_100] *= 1.3
    
    # 高段极端值
    high_mask = (binlab == "High")
    if high_mask.any():
        thr = np.percentile(ed[high_mask], 90)
        extreme = high_mask & (ed >= thr)
        w[extreme] *= 1.2
    
    return w
```

### 物理意义与设计逻辑

#### 分段定义 (基于能量密度 ED)

| 分段 | 范围 | 特征描述 |
|------|------|----------|
| **Low** | ED < 81 | 低能量密度区，致密度偏低 |
| **Mid** | 81 ≤ ED ≤ 111 | 过渡区，数据变化剧烈 |
| **High** | ED > 111 | 高能量密度区，性能趋于稳定 |

### 加权策略详解

#### ① 中段边界强化 (权重 ×1.3)
```python
near_76 = (binlab == "Mid") & (np.abs(ed - 80) <= 3)   # 77-83
near_100 = (binlab == "Mid") & (np.abs(ed - 110) <= 3)  # 107-113
```
- **物理意义**: 
  - 75-85 区间：低段→中段的**相变临界区**，材料性能快速提升
  - 105-115 区间：中段→高段的**饱和临界区**,致密度接近极限
- **为什么加权重**: 边界区域样本稀疏但信息量大，需要更多实验点来刻画非线性转变

#### ② 高段极端值强化 (权重 ×1.2)
```python
thr = np.percentile(ed[high_mask], 90)  # 高段 90% 分位数
extreme = high_mask & (ed >= thr)
```
- **统计意义**: 选取高段中**能量密度最高的 10%** 样本
- **工程价值**: 探索工艺极限条件，发现潜在的最优工艺窗口

#### ③ 权重叠加效应
```python
w = np.ones(len(candidate_df), dtype=float)  # 基础权重=1.0
w[near_76] *= 1.3    # 边界样本 → 1.3
w[extreme] *= 1.2    # 极端样本 → 1.2
# 若同时满足两个条件，权重可达 1.3×1.2 = 1.56
```

### 最终采集函数
```python
def value_function(models, X_candidates, candidate_df):
    base = committee_uncertainty(models, X_candidates)
    return base * boundary_weight(candidate_df)
```

**数学表达**:
$$
\text{score}_i = \underbrace{\text{Uncertainty}_i}_{\text{认知不确定性}} \times \underbrace{w_{\text{boundary},i}}_{\text{物理先验}}
$$

## 3️⃣ 采集函数 (Acquisition Function)

### 完整形式

您的主动学习框架使用的是 **加权委员会不确定性 (Weighted Committee Uncertainty, WCU)**:

$$
\alpha(\mathbf{x}_i) = \left[\sum_{t=1}^{T} w_t \cdot \text{Var}\left(\{f_m^{(t)}(\mathbf{x}_i)\}_{m=1}^M\right)\right] \times \beta(\mathbf{x}_i)
$$

其中:
- $f_m^{(t)}(\mathbf{x}_i)$: 第 m 个模型对第 t 个目标的预测
- $w_t$: 目标权重 (0.5, 0.3, 0.2)
- $\beta(\mathbf{x}_i)$: 边界加权系数 (1.0, 1.2, 或 1.3)

### 与其他采集函数的对比

| 采集函数 | 公式 | 特点 | 适用场景 |
|---------|------|------|---------|
| **EI (期望改进)** | $\mathbb{E}[\max(f(x)-f(x^*), 0)]$ | 平衡探索 - 利用 | 单目标贝叶斯优化 |
| **UCB (置信上界)** | $\mu(x) + \kappa\sigma(x)$ | 参数κ控制探索度 | 高斯过程 |
| **PI (概率改进)** | $P(f(x) > f(x^*) + \xi)$ | 保守策略 | 精细开发阶段 |
| **QBC (委员会查询)** | $\text{Var}(\{f_m(x)\})$ | 多模型投票 | **您的方法** ✅ |
| **WCU (加权 QBC)** | $\sum w_t \cdot \text{Var}_t \times \beta$ | **多目标 + 物理先验** | **您的创新** ✨ |

### 策略优势
🎯 **探索性强**: 直接选择模型分歧最大的区域，避免陷入局部最优  
🎯 **多目标平衡**: 通过权重矩阵协调多个性能指标的权衡  
🎯 **领域知识融合**: 边界加权引入材料科学先验知识  
🎯 **计算高效**: 只需 3 次前向传播，无需像 EI 计算积分

## 4️⃣ K-Means 多样性采样 (Diversity Sampling)

### 核心实现代码 (第 1064-1081 行)

```python
def diversify_batch(X_candidates, scores, quota, random_state=42):
    if quota <= 0:
        return []
    
    # 步骤 1: 按分数排序，选择 top-k 候选
    idx_sorted = np.argsort(scores)[::-1]
    top_k = max(quota * 2, quota)  # 扩大候选池至配额的 2 倍
    idx_top = idx_sorted[:min(top_k, len(idx_sorted))]
    X_top = X_candidates.iloc[idx_top]
    
    # 步骤 2: 在 Top 候选上聚类
    k = min(quota, len(X_top))
    km = KMeans(n_clusters=k, random_state=random_state, n_init=10).fit(X_top.values)
    
    # 步骤 3: 每簇选择最高分样本
    picks = []
    for c in range(k):
        cluster_idx = np.where(km.labels_ == c)[0]
        local_indices = idx_top[cluster_idx]
        local_scores = scores[local_indices]
        picks.append(local_indices[np.argmax(local_scores)])
    
    return picks
```

### 算法流程图

```
输入：候选池 X_candidates, 分数 scores, 配额 quota=2
│
├─ Step 1: 排序筛选
│  scores: [0.9, 0.8, 0.7, 0.6, 0.5, 0.4, 0.3, 0.2]
│  选 top_k = quota*2 = 4 个 → indices=[0,1,2,3]
│
├─ Step 2: K-Means 聚类 (k=quota=2)
│  X_top = X[0:4]
│  聚类结果：簇 0={0,2}, 簇 1={1,3}
│
└─ Step 3: 每簇选最高分
   簇 0: max(score[0]=0.9, score[2]=0.7) → 选 index=0
   簇 1: max(score[1]=0.8, score[3]=0.6) → 选 index=1
   输出：[0, 1]
```

### 关键设计亮点

#### ① 扩大候选池 (Exploration Buffer)
```python
top_k = max(quota * 2, quota)  # 2 倍配额
```
- **作用**: 在高分区域保持一定多样性，避免只选相似的样本
- **trade-off**: 太大→降低平均质量；太小→多样性不足

#### ② 聚类保证覆盖度
```python
km = KMeans(n_clusters=k, ...).fit(X_top.values)
```
- **特征空间**: 使用 4 维工艺参数 (功率、速度、间隙、层厚)
- **簇数设定**: `k = quota` (需要选几个点就分几类)
- **物理意义**: 确保选出的样本在工艺空间中分散分布

#### ③ 簇内择优 (Greedy Selection)
```python
picks.append(local_indices[np.argmax(local_scores)])
```
- **策略**: 每个簇内选分数最高的代表
- **效果**: 兼顾"高分"和"多样性"

### 与传统方法的对比

| 采样策略 | 方法 | 缺点 | 您的改进 |
|---------|------|------|---------|
| **纯随机** | `sample(n)` | 可能遗漏高分区 | ❌ 不使用 |
| **纯贪心** | `argsort()[:n]` | 样本高度相似 | ❌ 不使用 |
| **MMR (最大边际相关)** | $\max \lambda \cdot \text{score} - (1-\lambda) \cdot \text{similarity}$ | 需调参λ | ⚠️ 较复杂 |
| **K-Means 多样性** | 聚类后簇内择优 | **自动平衡探索 - 利用** | ✅ **您的选择** |

## 📈 整体主动学习循环架构

```
初始训练集 (X_train, y_train)
       ↓
训练 3 个模型 (RF/XGB/LGBM)
       ↓
计算采集函数 (WCU 分数)
       ↓
边界加权 (×1.2/1.3)
       ↓
K-Means 多样性采样
       ↓
选点：每段至少 1 个 (Low/Mid/High)
       ↓
添加新样本到训练集
       ↓
超参数优化 (贝叶斯搜索/BayesSearchCV)
       ↓
评估：R²/MAE/RMSE
       ↓
判断增益是否充足？
  ├─ 是 → 继续下一轮
  └─ 否 → 累计无改进轮数
       └─ ≥2 轮 → 停止
```

### 停止条件 (第 1294-1312 行)

```python
if len(history["R2_test"]) >= 2:
    r2_gain   = history["R2_test"][-1] - history["R2_test"][-2]
    mae_drop  = history["MAE_test"][-2] - history["MAE_test"][-1]
    rmse_drop = history["RMSE_test"][-2] - history["RMSE_test"][-1]
    
    insufficient = (
        (r2_gain  < 0.01) and   # R²提升<1%
        (mae_drop < 0.03) and   # MAE 下降<3%
        (rmse_drop < 0.03)      # RMSE 下降<3%
    )
    
    no_improve_rounds = no_improve_rounds + 1 if insufficient else 0

if no_improve_rounds >= 2:
    print("连续两轮增益不足，停止主动学习。")
```

**设计智慧**:
- ✅ **多指标联合判断**: 同时考虑 R²、MAE、RMSE
- ✅ **宽容策略**: 三个条件都未满足才判定为"不足"
- ✅ **缓冲轮次**: 允许连续 2 轮不改进，避免过早停止

## 🎯 总结与创新点

### 您的框架优势

1. **多目标不确定性量化**: 首次将 3 个材料性能指标通过权重矩阵融合
2. **物理先验引导**: 边界加权机制引入能量密度分段知识
3. **分段配额保障**: 确保 Low/Mid/High 每段至少 1 个样本，避免采样偏倚
4. **多样性 - 质量平衡**: K-Means 聚类解决"高分样本聚集"问题
5. **动态停止机制**: 基于性能增益的自适应终止策略

### 潜在改进方向

🔧 **不确定性校准**: 可使用 Conformal Prediction 校准置信区间  
🔧 **自适应权重**: 根据每轮表现动态调整 TARGET_WEIGHTS  
🔧 **混合采集函数**: 结合 EI 的利用能力，如 $\alpha = 0.7 \cdot \text{WCU} + 0.3 \cdot \text{EI}$  
🔧 **并行选点**: 使用 BatchMode 进一步优化多样性  
🔧 **替代超参数优化方法**: 可尝试 Optuna、Hyperopt 等更高效的贝叶斯优化器
